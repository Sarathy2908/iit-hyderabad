1. Project Overview
Intelli-Credit is a full-stack AI credit decisioning system designed for the Indian corporate lending context. It automates the end-to-end preparation of a Credit Appraisal Memo (CAM) by ingesting multi-format financial data, performing autonomous secondary research, and producing an explainable ML-based lending recommendation — all without relying on any third-party AI API.

CORE GOAL
Given a mid-sized Indian corporate borrower, produce a structured CAM with a lend/reject recommendation, a proposed credit limit, and a risk-adjusted interest rate — with full reasoning.

1.1 The Problem Being Solved
	•	Credit managers face a 'Data Paradox': massive data volumes but weeks-long processing times
	•	Manual process is slow, biased, and misses early warning signals in unstructured text
	•	Disparate data sources — GST, ITR, bank statements, annual reports, news — are never unified
	•	India-specific nuances (GSTR-2A vs 3B, CIBIL Commercial, MCA filings) are missed by generic tools

3. Module Breakdown
Module 1: Data Ingestor
PURPOSE
Parse and normalize all uploaded documents into a unified Firestore schema that downstream modules can consume.

1A. PDF Parser (Unstructured Documents)
Target files: Annual Reports, Legal Notices, Sanction Letters from other banks, Board Meeting Minutes, Rating Agency Reports.

Implementation approach (no external AI API):
	•	Use pdf-parse (Node.js) or pdfplumber (Python) to extract raw text from PDFs
	•	For scanned PDFs: run Tesseract OCR (open-source) via tesseract.js or pytesseract
	•	Text chunking: split extracted text into 512-token overlapping windows
	•	Keyword extraction: use a fine-tuned NER model (see ML Models section) to tag financial entities
	•	Pattern-based extractors for common fields: loan amounts, dates, party names, risk flags
	•	Regex patterns for INR amounts, CIN numbers, PAN, GSTIN
	•	Section detection: identify Balance Sheet, P&L, Notes to Accounts by heading patterns
	•	Table extraction: use camelot-py or pdfplumber's table mode for financial tables

1B. GST Return Analyser
Target files: GSTR-3B (self-declared), GSTR-2A (auto-populated from counterparties), GSTR-1.

	•	Parse GSTR JSON/XML exports from the GST portal
	•	Cross-check GSTR-3B reported turnover vs GSTR-2A matched purchases — flag discrepancies > 15%
	•	Detect circular trading: identify if the same GSTIN appears as both buyer and seller across periods
	•	Revenue inflation signals: sudden turnover spikes not matched by ITC growth
	•	Compute effective tax rate and flag if it deviates from sector norms stored in Firestore

INDIA CONTEXT
GSTR-2A is auto-drafted from supplier filings and represents actual purchases. GSTR-3B is self-declared. A large gap between them is a red flag for fake invoicing or circular trading.

1C. Bank Statement Analyser
Target files: 12-24 months of bank statements in CSV or PDF format.

	•	Parse CSV exports directly; for PDF statements, use Tabula-py for table extraction
	•	Compute: average monthly balance, inflow-outflow ratio, EMI obligations, bounce frequency
	•	Flag: cheque returns, unsanctioned overdrafts, large round-number transactions (structuring signals)
	•	Cross-verify bank credits against GST turnover — a major divergence is a fraud signal
	•	Classify transaction categories using a simple keyword-based classifier (no ML needed here)

1D. Structured Financial Statement Parser
Target files: Audited Balance Sheet, P&L Statement, Cash Flow Statement (PDF or structured Excel).

	•	Extract key ratios automatically: Current Ratio, Debt-to-Equity, DSCR, ICR, ROCE
	•	Compute YoY growth for revenue, EBITDA, PAT — flag declining trends
	•	Identify contingent liabilities, related-party transactions, qualified audit opinions
	•	Store normalized financials in Firestore as a time-series array per company


Module 2: Research Agent (Digital Credit Manager)
PURPOSE
Autonomously gather external intelligence and integrate qualitative primary observations into the credit assessment.

2A. Web Crawler & Secondary Research Engine
The Research Agent runs as a Cloud Function triggered after document ingestion. It uses Node.js with Puppeteer (headless Chrome) or Cheerio for lightweight scraping.

	•	Company-level research: search news for promoter names, company name, CIN
	•	Sources: Google News RSS, Economic Times, Business Standard, Moneycontrol
	•	Classify headlines as: Positive / Neutral / Negative / Regulatory Risk
	•	MCA portal scraper: fetch ROC filings, director details, charge satisfaction status
	•	e-Courts portal: check for litigation against promoters or the company
	•	CIBIL Commercial (if accessible): parse credit summary report for defaults, write-offs
	•	Sector research: fetch RBI circulars, SEBI notices for the company's industry
	•	All findings stored in Firestore with source URL, date, and sentiment score

TECH NOTE
Use Puppeteer running in a Cloud Function (or Cloud Run for longer tasks). Set a 60-second timeout. Cache results in Firestore for 24 hours to avoid re-scraping on every request.

2B. Qualitative Notes Portal
A dedicated Vue component in the frontend allows the Credit Officer to input primary due diligence observations.

	•	Structured form with dropdowns + free-text fields:
	•	Factory utilisation capacity: 0-25% / 25-50% / 50-75% / 75-100%
	•	Management quality: Excellent / Adequate / Weak / Concerns noted
	•	Collateral condition: Prime / Standard / Sub-standard
	•	Free-text: key observations from site visit and management interview
	•	Each observation maps to a pre-defined score adjustment delta stored in Firestore config
	•	Example: factory at 40% capacity subtracts 8 points from the capacity score
	•	Credit Officer can attach photos (stored in Cloud Storage)

2C. News Sentiment Classifier
A lightweight ML model (see ML section) classifies scraped news headlines as positive, negative, neutral, or regulatory-risk. The aggregate sentiment feeds into the final risk score.

3B. Credit Scoring Model
The model produces three outputs: a credit score (0-100), a recommended loan limit, and a risk-adjusted interest rate spread.

	•	Base model: Gradient Boosted Trees (XGBoost) trained on synthetic Indian corporate loan data
	•	Calibration: Platt scaling to convert raw score to default probability
	•	Limit formula: min(25% of tangible net worth, 3x EBITDA) adjusted by score band
	•	Rate formula: base rate + risk premium where premium = f(score, sector, collateral type)
	•	SHAP values computed for every prediction to generate the explainability narrative

3C. CAM Document Generator
Generates a professional Credit Appraisal Memo as a Word or PDF document.

	•	Template engine: Docxtemplater (Node.js) fills a pre-designed DOCX template
	•	Sections: Executive Summary, Five Cs Analysis, Financial Highlights, Risk Flags, Recommendation
	•	Each section auto-populated from Firestore data and SHAP explanations
	•	Risk flags section explicitly calls out: litigation risk, circular trading, GST discrepancies
	•	Final document uploaded to Cloud Storage; download link sent via Firestore update

EXPLAINABILITY REQUIREMENT
The CAM must include a 'Decision Rationale' box that says, for example: 'Rejected due to high litigation risk (3 active cases found) and GSTR-2A vs 3B discrepancy of 28%, despite strong ICR of 2.4x.' SHAP drives this narrative.


4. Custom ML Models (No External API)
CONSTRAINT
All ML inference must run within the Firebase project — no calls to OpenAI, Gemini, Claude, or any external AI API. Models are trained offline and served via Cloud Run or Cloud Functions.

Model 1: Financial NER (Named Entity Recognition)
Attribute
Detail
Purpose
Extract financial entities from PDF text: amounts, dates, party names, account numbers, loan terms
Architecture
BiLSTM-CRF or fine-tuned DistilBERT (small, 66M params) for sequence labeling
Training Data
Synthetically generated Indian financial documents + publicly available SEBI filings
Entity Types
ORG, PERSON, AMOUNT_INR, DATE, GSTIN, PAN, CIN, LOAN_TERM, COLLATERAL_TYPE
Inference
Python Flask endpoint on Cloud Run; called by Data Ingestor Cloud Function
Output
JSON: {entity_type, text, start_char, end_char, confidence}
Model Size
~250MB (DistilBERT) or ~15MB (BiLSTM-CRF) — fits in Cloud Run easily
Framework
HuggingFace Transformers (offline) or spaCy with custom NER pipeline

Model 2: News Sentiment Classifier
Attribute
Detail
Purpose
Classify news headlines and snippets about the borrower into sentiment categories
Architecture
Fine-tuned DistilBERT-multilingual (handles Hindi + English headlines) or TF-IDF + Logistic Regression for speed
Classes
POSITIVE / NEUTRAL / NEGATIVE / REGULATORY_RISK / LITIGATION
Training Data
Labelled financial news headlines from ET, BS, Moneycontrol (manually labelled or weakly supervised)
Inference
Batched — process all scraped headlines in one call to Cloud Run
Output
Per-headline label + confidence; aggregate company-level sentiment score (weighted average)
Model Size
~250MB (DistilBERT) or ~5MB (TF-IDF + LR) — use TF-IDF version for hackathon speed
Fallback
If confidence < 0.6, classify as NEUTRAL and flag for human review

Model 3: Credit Scoring & Recommendation Model
Attribute
Detail
Purpose
Predict probability of default + output credit limit and rate recommendation
Architecture
XGBoost Classifier (default probability) + rule-based limit/rate calculator on top of score
Training Data
Synthetic dataset of 50,000 corporate loan cases generated from RBI published NPA data distributions
Features
~35 features across 5 Cs (see Feature Engineering above)
Explainability
SHAP TreeExplainer — produces per-feature contribution values for each prediction
Output
{ score: 72, default_prob: 0.12, recommendation: APPROVE, limit_cr: 15, rate_spread_bps: 180, shap_values: {...} }
Calibration
Isotonic regression for probability calibration post-training
Model Size
~10MB — can run directly in a Cloud Function (no Cloud Run needed)
Framework
XGBoost (Python), SHAP library, joblib for serialization
Serving
Model pickle file stored in Cloud Storage; loaded on Cloud Function cold start and cached

Model 4: Document Table Extractor
Attribute
Detail
Purpose
Extract structured financial tables from scanned PDFs (Balance Sheet, P&L)
Architecture
Rule-based heuristics using pdfplumber + Camelot-py for digital PDFs; Tesseract OCR + custom table detector for scanned docs
Table Detector
Simple CNN trained on document images to classify: IS_TABLE vs IS_TEXT region (can use a lightweight MobileNet variant)
Output
Pandas DataFrame per detected table; rows and columns normalized to standard INDAS schema
Post-processing
Map extracted row labels to canonical names: 'Total Revenue' = 'Revenue from Operations + Other Income'
Framework
Camelot, pdfplumber, Tesseract, OpenCV for image preprocessing
Model Size
Table detector CNN: ~5MB; rest is rule-based with no ML model file

6.3 Cloud Run — ML Model Server
Python-based ML models that are too large for Cloud Functions (>10MB, long inference) run as a Flask app on Cloud Run within the same GCP project:

	•	Container: python:3.11-slim with Flask, XGBoost, SHAP, HuggingFace Transformers, Tesseract
	•	Endpoints: POST /ner, POST /sentiment, POST /score, POST /extract-tables
	•	Auth: requests from Cloud Functions use GCP service account identity tokens — no API key needed
	•	Scaling: Cloud Run min-instances=1 to avoid cold starts during demo
	•	Model files: downloaded from Cloud Storage on container startup and cached in /tmp
	•	Dockerfile and Cloud Run config stored in /ml directory within the monorepo

COST NOTE
Cloud Run charges per request with a generous free tier. For the hackathon, set max-instances=2 and a 10-second timeout. The XGBoost model scores in <100ms; NER in <500ms per document.

9. AI IDE Prompt Document
Use the following prompts when working with an AI coding assistant (Cursor, GitHub Copilot, etc.) to guide its reasoning. These are structured as system-level context injections.

9.1 Global Context Prompt
You are building Intelli-Credit: an AI-powered corporate credit appraisal system for Indian banks.
Stack: Vue 3 (Composition API, Pinia, Vue Router, Tailwind) + Firebase (Hosting, Firestore, Cloud Functions, Cloud Storage, Auth) + Python Cloud Run for ML models.
HARD CONSTRAINT: Do NOT use any external AI API (no OpenAI, Gemini, Claude, Cohere, etc.). All ML must be custom-built using XGBoost, spaCy, HuggingFace Transformers (offline), or scikit-learn.
India-specific context: understand GSTR-2A vs GSTR-3B, CIN, GSTIN, PAN, MCA filings, CIBIL Commercial scores, SARFAESI act, RBI prudential norms, IndAS accounting standards.
Explainability is mandatory: every model output must include a plain-English reason string. Use SHAP for the scoring model.

9.2 Module-Specific Prompts
Data Ingestor Prompt
Build a Node.js Cloud Function triggered on Cloud Storage file upload. Route files by extension:
  - .pdf → call Python /extract endpoint on Cloud Run (pdfplumber + Tesseract if scanned)
  - .json / .xml → GST analyser (parse GSTR-3B and GSTR-2A, compute gap, detect circular trading)
  - .csv → bank statement parser (compute avg balance, EMI burden, bounce frequency)
Write extracted data to Firestore under /cases/{caseId}/financials and update case status field.

Scoring Model Prompt
Build a Python XGBoost classifier for corporate credit default prediction.
Input: 35-feature vector covering the Five Cs (Capacity, Capital, Character, Collateral, Conditions) + GST quality + banking behaviour + primary qualitative observations.
Output: { score (0-100), default_probability, recommendation (APPROVE/REJECT/REFER), credit_limit_cr, rate_spread_bps, shap_values }.
Use SHAP TreeExplainer. Generate a plain-English reason string: list the top 3 positive and top 3 negative SHAP contributors.
Expose as POST /score on Flask. Accept JSON feature vector. Return full result JSON.

Vue Frontend Prompt
Build a Vue 3 SPA with Pinia and VueFire. All state comes from Firestore real-time listeners.
Key components: UploadZone (multi-file drag-drop to Cloud Storage), CaseStatusTracker (real-time pipeline steps), ScoreGauge (animated 0-100 gauge), ShapWaterfall (SHAP bar chart using Chart.js), QualitativeForm (structured Credit Officer notes form), CAMPreview (PDF viewer).
Use Tailwind CSS only. No component library dependencies unless explicitly approved. Keep bundle size small for Firebase Hosting.
